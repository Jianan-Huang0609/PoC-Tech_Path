## 开发规范

### 提交格式
```
<type>(<scope>): <subject>

<body>

<footer>
```

### 类型说明
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

### 示例
```
feat(matrix): 添加技术栈矩阵功能
fix(storage): 修复数据保存问题
docs(readme): 更新项目说明文档
```
