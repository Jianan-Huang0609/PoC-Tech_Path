## 🤝 贡献指南

### 提交规范
遵循 [Conventional Commits](https://conventionalcommits.org/) 规范：

```
<type>(<scope>): <subject>

<body>

<footer>
```

### 类型说明
- `feat`: 新功能
- `fix`: 修复bug
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建过程或辅助工具的变动

### 版本发布流程
1. 更新版本号 (`package.json`)
2. 更新更新日志 (`update.md`)
3. 更新Git记录 (`Git_Update.md`)
4. 创建Git标签
5. 推送到远程仓库
